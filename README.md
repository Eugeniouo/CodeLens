# CodeLens — Умный поиск по кодовой базе

CodeLens — это прототип RAG-системы для семантического поиска и архитектурного аудита исходного кода. Система позволяет разработчикам находить нужные фрагменты кода (функции и классы), формулируя запросы на естественном языке (русском или английском), даже если слова в запросе не совпадают с названиями переменных или функций.

---

## Архитектура системы

Проект состоит из четырех основных компонентов, полностью реализованных на Python 3.12:

1. **Модуль парсинга и векторизации (`parser.py`, `parser_java.py`, `embedder.py`):**
   * **Парсинг:** Используется встроенный модуль `ast` для Python и библиотека `tree-sitter` для Java. Код разбирается на абстрактные синтаксические деревья (AST), из которых извлекаются классы, методы и их docstring.
   * **Векторизация:** Модель `intfloat/multilingual-e5-small` преобразует текст и код в эмбеддинги. В тексты автоматически добавляются префиксы `passage:` и `query:` для обеспечения максимального качества семантического поиска.
2. **Векторная база данных (ChromaDB):**
   * Локальное хранилище векторных представлений и метаданных (путь к файлу, тип сущности, имя функции/класса, номера строк, javadoc/docstring). Поиск выполняется по косинусному расстоянию.
3. **Веб-интерфейс (Streamlit):**
   * Полнофункциональное B2B-приложение для взаимодействия с системой без необходимости использования сторонних frontend-фреймворков.
4. **Аналитический LLM-модуль (`llm.py`):**
   * Интеграция с моделями через OpenAI-совместимый API (Groq) для генерации связных ответов и объяснения логики найденного кода в потоковом режиме (streaming).

## Обоснование стратегии чункования

**Мы используем семантическое разбиение на основе AST-деревьев (1 чанк = 1 функция или 1 класс).**

В отличие от «наивного» текстового разбиения (например, по фиксированному числу символов или токенов), подход на основе AST гарантирует, что логический контекст исполнения остается неразрывным.

**Почему это важно:**
1. **Сохранение семантики:** Функция или класс — это минимальная, атомарная и законченная единица бизнес-логики.
2. **Точность векторизации:** Сигнатура, docstring и тело функции попадают в один эмбеддинг, что кардинально повышает метрику Precision@5. Поисковая модель оценивает контекст операции целиком, а не случайный обрывок синтаксической конструкции.
3. **Читаемость результатов:** Пользователь получает в интерфейсе готовый к анализу блок кода, а не случайный срез строк.

## Возможности веб-интерфейса Streamlit

Интерфейс разделен на три функциональных модуля:
* **🔍 Семантический поиск:** Строка ввода запросов, настройка работы с LLM-ассистентом и интерактивная выдача карточек с кодом. Система автоматически определяет язык файла (Python/Java) и применяет соответствующую подсветку синтаксиса.
* **⚙️ Индексация базы:** Модуль управления векторной базой данных прямо из UI. Позволяет указать путь к новому репозиторию, выбрать режим полной очистки базы (Reset) и запустить синтаксический анализ и векторизацию в один клик.
* **📊 Валидация системы:** Страница автоматического тестирования поискового движка на валидационном датасете `eval_questions.json` с визуализацией метрик Precision@5, MRR и средней задержки (Latency).

## Варианты запуска системы

Вы можете запустить систему как локально, так и в изолированном Docker-окружении.

### Вариант 1. Запуск через Docker Compose (Рекомендуемый)

Проект полностью контейнеризирован. Конфигурация включает автоматическую сборку компиляторов (gcc, g++) для работы `tree-sitter` внутри Linux-контейнера.

1. Убедитесь, что ваш API-ключ указан в файле `.env` в корне проекта.
2. Запустите сборку и старт приложения одной командой:
   ```bash
   docker compose up --build

```

3. Откройте в браузере: `http://localhost:8501`

*База данных ChromaDB монтируется как volume, поэтому ваши проиндексированные данные сохраняются при перезапуске контейнеров.*

### Вариант 2. Локальный запуск

1. Установите зависимости:
```bash
pip install -r requirements.txt

```


2. **Индексирование через CLI** (также доступно в веб-интерфейсе):
```bash
python index.py <путь_к_папке_с_кодом>

```


3. **Запуск UI:**
```bash
streamlit run app.py

```

## Примеры запросов (Топ-5)

Система является полностью мультиязычной и обрабатывает запросы на русском и английском языках:

1. **Запрос:** *"Как здесь обрабатываются ошибки авторизации?"*
* **Результат:** Находит middleware проверки JWT-токенов или функции обработки HTTP 401/403.


2. **Запрос:** *"Где реализована генерация JWT токена и сроки его жизни?"*
* **Результат:** Возвращает функции создания сессий и утилиты криптографии с параметрами `expires_delta`.


3. **Запрос:** *"Подключение к базе данных и настройки пула соединений"*
* **Результат:** Идентифицирует конфигурационные файлы и фабрики сессий SQLAlchemy или методы `get_db()`.


4. **Запрос:** *"Алгоритм расчета финальной стоимости корзины с учетом скидок"*
* **Результат:** Находит сервисный слой корзины (например, метод `calculate_total` или применение промокодов).


5. **Запрос (Eng):** *"How is the user profile updated in the database?"*
* **Результат:** Успешно находит CRUD-операции обновления сущности пользователя (`update_user`), демонстрируя работу кросс-языкового поиска.
